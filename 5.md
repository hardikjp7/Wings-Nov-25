## **Topic 17: Dual Axis & Combined Charts**

### **What is a Dual Axis Chart?**

**Dual Axis** = Two measures plotted on the same chart with **separate Y-axes** (left and right)

**Why use dual axis?**
âœ… Compare measures with different scales
âœ… Show correlation between two metrics
âœ… Overlay different mark types (bars + lines)

**Example Use Cases:**
- Sales (bars) and Profit Margin (line) over time
- Actual vs Target comparison
- Temperature and Rainfall on same chart
- Quantity sold (bars) with Price (line)

---

### **Creating a Dual Axis Chart**

**Method 1: Drag to Opposite Axis**
1. Drag first measure to Rows
2. Drag second measure to Rows (creates two separate charts)

**Method 2: Drag and Drop**
1. Create chart with first measure
2. Drag second measure and drop on right side of view (when you see ruler icon)
3. Automatic dual axis

**Method 3: Ctrl+Drag (Duplicate)**
1. Place measure on Rows
2. Ctrl+Drag same measure to Rows (duplicates it)

4. Edit one to different aggregation/calculation

### **Synchronizing Axes** â­ **IMPORTANT!**

**Problem:** By default, each axis has independent scale
```
Left axis: 0 - 100,000 (Sales)
Right axis: 0 - 20,000 (Profit)
```
Can visually mislead!

**Solution: Synchronize Axes**
1. Right-click on one axis â†’ **Synchronize Axis**
2. Both axes now share same scale (0 - 100,000)

**When to synchronize:**
- Measures in same units (both currency)
- Want true visual comparison
- Avoid misleading viewers

**When NOT to synchronize:**
- Different units (Temperature Â°F vs Rainfall inches)
- Vastly different scales (one is millions, other is percentages)
- Want to see both trends clearly

### **Editing Individual Measures on Dual Axis**

**Each measure has separate Marks Card:**

**Top Marks Card** = First measure (SUM(Sales))
- Can set to Bar, Line, Circle, etc.
- Independent color, size, label

**Bottom Marks Card** = Second measure (SUM(Profit))
- Different mark type
- Different formatting

**"All" Marks Card** = Applies to both

### **Practical Example: Sales vs Profit Margin**

**Step-by-step:**

1. **Drag Order Date to Columns** (continuous, by month)
2. **Drag Sales to Rows** â†’ Creates line chart
3. **Create calculated field: Profit Margin**
   ```
   SUM([Profit]) / SUM([Sales])
   ```
4. **Drag Profit Margin to Rows** â†’ Second chart below
5. **Right-click Profit Margin pill â†’ Dual Axis**
6. **Format:**
   - SUM(Sales) Marks Card â†’ Bar, Blue color
   - AGG(Profit Margin) Marks Card â†’ Line, Orange color
   - Format Profit Margin axis as Percentage
7. **Optional:** Don't synchronize (Sales in $, Margin in %)

**Result:** Bars show sales volume, line shows profitability trend

### **Blended Axis vs Dual Axis**

**Dual Axis:**
- Two separate Y-axes (left and right)
- Can have different scales
- Measures can be in different units

**Blended Axis (Measure Names/Values):**
- Single shared axis
- All measures must use same scale
- Simpler for similar measures

We'll cover Measure Names/Values approach later in this topic.

---

### **Dual Axis Best Practices**

âœ… **DO:**
- Use different mark types for clarity (bar + line)
- Label axes clearly
- Synchronize when measures comparable
- Use color to distinguish measures
- Add legend

âŒ **DON'T:**
- Mislead with unsynchronized axes of same unit
- Overcomplicate with too many overlays
- Use when simple side-by-side is clearer
- Forget to label which axis is which

**Different from Dual Axis:**
- Single shared axis
- All measures same scale
- Cleaner when measures similar

### **Advanced: Bullet Graph** â­ **Common in Business**

**Bullet Graph** = Compact way to show Actual vs Target with ranges

**Components:**
- Bar (Actual value)
- Line (Target/Goal)
- Background shading (Good/Fair/Poor ranges)

**How to create:**
1. Place dimension on Rows
2. Place Actual on Columns (Bar)
3. Place Target on Columns (Dual axis)
4. Change Target to Line
5. Add reference distribution for ranges (next topic)

**Or use Show Me:**
- Select dimension + 2 measures
- Click Bullet Graph in Show Me

---

### **Advanced: Gantt Chart with Duration**

**Gantt Chart** = Timeline visualization (project management)

**Setup:**
1. Drag Start Date to Columns (continuous)
2. Drag Task Name to Rows
3. Change mark type to **Gantt Bar**
4. Drag Duration to Size
5. Optional: Color by Status

**Dual Axis Enhancement:**
- Add milestones as second measure
- Show as shapes on timeline

### **Troubleshooting Dual Axis**

**Problem: "Can't see second measure"**
- Solution: Check mark types - might be hidden behind
- Try line + bar instead of bar + bar

**Problem: "Dual Axis option grayed out"**
- Solution: Must have 2+ measures on Rows/Columns
- Can't dual axis with dimensions

**Problem: "Axes showing weird scales"**
- Solution: Edit axis â†’ Fixed start/end
- Or synchronize if appropriate

**Problem: "Can't tell measures apart"**
- Solution: Use different colors, mark types
- Add labels, legend

### **What are Reference Lines?**

**Reference Line** = Visual indicator showing a specific value or statistical measure

**Types:**
- **Line** - Horizontal or vertical line at specific value
- **Band** - Shaded area between two values
- **Distribution** - Shows data spread (confidence intervals, percentiles)
- **Box Plot** - Statistical summary

**Common uses:**
âœ… Show targets, goals, thresholds
âœ… Indicate averages, medians
âœ… Highlight acceptable ranges
âœ… Show statistical distributions

---

### **Trend Line Statistics**

**Hover over trend line:**

- **Equation** - y = mx + b (for linear)
- **R-squared** - Goodness of fit (0-1, higher = better)
- **P-value** - Statistical significance

**Example:**

### **R-squared Interpretation** â­ **EXAM IMPORTANT!**

**RÂ² = 1.0** â†’ Perfect fit (100% variance explained)
**RÂ² = 0.8** â†’ Good fit (80% variance explained)
**RÂ² = 0.5** â†’ Moderate fit
**RÂ² = 0.2** â†’ Weak fit
**RÂ² = 0.0** â†’ No linear relationship

**P-value:**
- **P < 0.05** â†’ Statistically significant trend
- **P > 0.05** â†’ May be random, not reliable
