### **Types of Calculations:**

#### **1. Row-Level Calculations**

Calculated **for each row** in your data before aggregation.

**Example 1: Profit Margin (per row)**
```
[Profit] / [Sales]
```
- Calculates margin for **each transaction**
- Then aggregated in view (AVG, SUM, etc.)

**Example 2: Full Name**
```
[First Name] + " " + [Last Name]
```
- Concatenates for each customer record

**Example 3: Discount Amount**
```
[Sales] * [Discount]
```
- Calculates discount dollars per order

**Key characteristic:** References fields WITHOUT aggregation functions (SUM, AVG, etc.)

---

#### **2. Aggregate Calculations**

Calculated **after** data is aggregated in view.

**Example 1: Overall Profit Margin**
```
SUM([Profit]) / SUM([Sales])
```
- First sums all profit, sums all sales
- Then divides
- **Different result than row-level!**

**Example 2: Average Order Value**
```
SUM([Sales]) / COUNT([Order ID])
```
- Total sales divided by number of orders

**Example 3: Sales Per Customer**
```
SUM([Sales]) / COUNTD([Customer ID])
```
- COUNTD = Count Distinct

**Key characteristic:** Uses aggregation functions (SUM, AVG, COUNT, etc.)

---

### **Row-Level vs Aggregate - CRITICAL DIFFERENCE!**

**Example to understand:**

**Data:**
```
Order | Sales | Profit
1     | 100   | 20      â†’ Margin: 20%
2     | 200   | 30      â†’ Margin: 15%
3     | 300   | 60      â†’ Margin: 20%
```

**Row-Level Calculation:**
```
[Profit] / [Sales]
```
**Process:**
1. Calculate margin for each row: 20%, 15%, 20%
2. When aggregated as AVG: **(20% + 15% + 20%) / 3 = 18.3%**

**Aggregate Calculation:**
```
SUM([Profit]) / SUM([Sales])
```
**Process:**
1. Sum all profit: 20 + 30 + 60 = 110
2. Sum all sales: 100 + 200 + 300 = 600
3. Divide: **110 / 600 = 18.3%**

**In this case, same result! But not always...**

---

**Different Example:**

**Data:**
```
Order | Sales | Profit
1     | 100   | 20      â†’ Margin: 20%
2     | 200   | 60      â†’ Margin: 30%
```

**Row-Level (then averaged):**
```
AVG([Profit] / [Sales])
= (20% + 30%) / 2 
= 25%
```

**Aggregate:**
```
SUM([Profit]) / SUM([Sales])
= (20 + 60) / (100 + 200)
= 80 / 300
= 26.67%
```

**Different results!** ğŸ¯

**Which is correct?**
- Depends on business question!
- **Aggregate usually correct** for overall metrics
- Row-level for analyzing individual transaction patterns

**EXAM LOVES THIS!** Questions like: "Why do two profit margin calculations show different results?"

### **Order of Operations:**

Tableau follows standard math order: **PEMDAS**

1. **P**arentheses `()`
2. **E**xponents `^`
3. **M**ultiplication `*` and **D**ivision `/` (left to right)
4. **A**ddition `+` and **S**ubtraction `-` (left to right)

**Example:**
```
[Sales] - [Cost] * [Tax Rate]
```
- Multiplies first: `[Cost] * [Tax Rate]`
- Then subtracts from Sales

**To change order, use parentheses:**
```
([Sales] - [Cost]) * [Tax Rate]
```
- Subtracts first
- Then multiplies by tax rate

### **Handling NULL Values:**

**NULL** = Missing/undefined value

**Problem:** Math with NULL = NULL
```
[Sales] + [Bonus]
If Bonus is NULL â†’ Result is NULL
```

**Solution: ZN() function** (Zero if Null)
```
[Sales] + ZN([Bonus])
If Bonus is NULL â†’ Treats as 0
```

**Other NULL functions:**
- `IFNULL([Field], replacement)` - Replace NULL with specific value
- `ISNULL([Field])` - Returns TRUE if NULL
- `ZN([Field])` - Returns 0 if NULL (shortcut)
